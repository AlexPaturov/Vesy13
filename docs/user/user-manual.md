# Руководство пользователя Vesy13

## 1. Назначение

Vesy13 - программа для работы с вагонными весами N13. Она принимает показания
весового оборудования, выполняет статическое или динамическое взвешивание
вагонов, сохраняет результаты, переносит их в учетную систему, позволяет
корректировать записи и формирует PDF-отвесные.

Документ предназначен для оператора весов, старшего оператора и администратора.

## 2. Роли пользователей

### Оператор

Оператор выполняет ежедневную работу:

- проверяет связь с АЦП;
- выполняет статическое или динамическое взвешивание;
- завершает состав;
- переносит результаты в учетную систему;
- печатает отвесные;
- сообщает о сбоях старшему оператору или администратору.

### Старший оператор

Старший оператор дополнительно:

- проверяет спорные записи;
- исправляет реквизиты в разделе `Корректировки`;
- контролирует правильность переноса;
- просматривает журнал событий.

### Администратор

Администратор отвечает за:

- настройку COM-порта;
- калибровку;
- изменение лимитов и дискретности;
- смену пароля;
- разбор ошибок оборудования и баз данных.

Не меняйте сервисные настройки и калибровку без разрешения ответственного лица.

## 3. Основные понятия

| Термин | Значение |
| --- | --- |
| АЦП | Устройство, которое передает программе сырые показания весов |
| Тележка | Часть вагона, которая фиксируется отдельно при взвешивании |
| Состав | Последовательность вагонов, взвешиваемых в рамках одной операции |
| Брутто | Полный вес вагона с грузом |
| Тара | Вес пустого вагона |
| Нетто | Вес груза, обычно `Брутто - Тара` |
| Приход `GPRI` | Приходное направление учетной системы |
| Расход `GRAS` | Расходное направление учетной системы |

## 4. Главное окно

После запуска открывается главное меню.

| Раздел | Назначение |
| --- | --- |
| `Статическое взвешивание` | Взвешивание неподвижного вагона |
| `Динамическое взвешивание` | Взвешивание вагона в движении |
| `Сервис` | Настройки, связь, каналы, калибровка |
| `Корректировки` | Перенос в учетную систему и исправление записей |
| `Печать` | Поиск записей и формирование отвесной |
| `Журнал` | Просмотр действий и ошибок |

Внизу окна отображается состояние АЦП:

- зеленый индикатор - оборудование подключено;
- красный индикатор - связи с оборудованием нет.

Не начинайте взвешивание при красном индикаторе.

## 5. Подготовка к смене

### Проверка рабочего места

1. Убедитесь, что компьютер включен и программа запущена.
2. Убедитесь, что весовое оборудование включено.
3. Проверьте кабель связи с АЦП.
4. Убедитесь, что весы свободны, если требуется установка нуля.
5. Проверьте, что в главном окне индикатор АЦП зеленый.

### Проверка связи

1. Откройте `Сервис`.
2. Перейдите к мониторингу.
3. Убедитесь, что выбран правильный COM-порт.
4. Если связи нет, нажмите `Подключить`.
5. При необходимости обновите список портов.
6. Вернитесь в главное окно.

### Проверка нуля

1. Откройте нужный режим взвешивания.
2. Убедитесь, что на весах нет вагона.
3. Посмотрите текущее значение.
4. Если значение отличается от нуля в допустимых пределах, нажмите `Ноль`.
5. Если программа не разрешает установить ноль, сообщите старшему оператору или администратору.

### Когда нельзя начинать работу

Не начинайте взвешивание, если:

- индикатор АЦП красный;
- программа показывает ошибку подключения;
- весы заняты, а требуется установка нуля;
- не выбрано направление для динамического взвешивания;
- есть сомнения в правильности калибровки;
- база данных недоступна и результаты не сохраняются.

## 6. Статическое взвешивание

Статический режим используется, когда вагон останавливается на весах.

### Элементы окна

| Элемент | Назначение |
| --- | --- |
| Большое значение веса | Текущий вес в тоннах |
| `Тележка 1` | Зафиксированный вес первой тележки |
| `Тележка 2` | Зафиксированный вес второй тележки |
| `ВЗВЕСИТЬ [Пробел] - Тележка 1` | Фиксация первой тележки |
| `ВЗВЕСИТЬ [Пробел] - Тележка 2` | Фиксация второй тележки |
| `Ноль` | Установка текущего значения как нуля |
| `Завершить состав` | Завершение текущего состава |
| Таблица последних строк | Последние взвешенные вагоны |

### Порядок взвешивания одного вагона

1. Откройте `Статическое взвешивание`.
2. Убедитесь, что индикатор АЦП зеленый.
3. Установите вагон так, чтобы на весах была первая тележка.
4. Дождитесь стабильных показаний.
5. Нажмите `ВЗВЕСИТЬ` или клавишу `Пробел`.
6. Убедитесь, что поле `Тележка 1` заполнено.
7. Установите вагон так, чтобы на весах была вторая тележка.
8. Дождитесь стабильных показаний.
9. Нажмите `ВЗВЕСИТЬ` или клавишу `Пробел`.
10. Проверьте строку вагона в таблице.

После второго нажатия программа рассчитает вес обеих тележек, общий вес вагона,
присвоит порядковый номер и сохранит запись в локальную базу.

### Взвешивание состава

1. Выполните взвешивание первого вагона.
2. Повторите процедуру для каждого следующего вагона.
3. Следите за порядковыми номерами.
4. После последнего вагона нажмите `Завершить состав`.
5. Подтвердите завершение, если программа запросит подтверждение.

После завершения состава счетчик вагонов сбрасывается.

### Закрытие окна во время взвешивания

Если первая тележка уже зафиксирована, а вторая еще нет, программа предупредит о
незавершенном вагоне. При подтверждении выхода замер первой тележки будет
потерян.

## 7. Динамическое взвешивание

Динамический режим используется, когда вагон проходит через весы в движении.

### Элементы окна

| Элемент | Назначение |
| --- | --- |
| `Направление движения состава` | Выбор направления `-> (+)` или `<- (-)` |
| Большое значение веса | Текущий расчетный вес |
| Статус | Подсказка текущего этапа |
| `ВЗВЕСИТЬ [Пробел]` | Фиксация тележки |
| `Ноль` | Установка нуля |
| `Завершить состав` | Завершение состава |
| Таблица | Последние результаты |

### Выбор направления

Перед началом проверьте направление движения состава:

- `-> (+)` - движение в направлении плюс;
- `<- (-)` - движение в направлении минус.

Направление влияет на расчет веса. Ошибка направления может привести к
неправильному результату.

### Порядок взвешивания одного вагона

1. Откройте `Динамическое взвешивание`.
2. Убедитесь, что индикатор АЦП зеленый.
3. Выберите правильное направление движения.
4. При прохождении первой тележки нажмите `ВЗВЕСИТЬ` или `Пробел`.
5. Убедитесь, что статус изменился на ожидание второй тележки.
6. При прохождении второй тележки нажмите `ВЗВЕСИТЬ` или `Пробел`.
7. Проверьте строку в таблице.

После второго нажатия программа сохраняет вагон с режимом `ДИНАМИКА` и выбранным
направлением.

### Типовые ошибки динамического режима

| Ошибка | Последствие | Что делать |
| --- | --- | --- |
| Выбрано неверное направление | Вес рассчитан по неверному коэффициенту | Не переносить запись, сообщить старшему оператору |
| Нажатие сделано слишком рано | Зафиксирован неверный код АЦП | Проверить запись, при необходимости повторить взвешивание |
| Нажатие сделано слишком поздно | Вес может быть некорректным | Проверить запись, при необходимости повторить взвешивание |
| Пропущена тележка | Вагон не завершен корректно | Действовать по регламенту участка и сообщить старшему оператору |

## 8. Установка нуля

Кнопка `Ноль` устанавливает текущее значение как нулевое смещение.

### Когда устанавливать ноль

Устанавливайте ноль:

- перед началом смены;
- после проверки пустых весов;
- после указания старшего оператора;
- если пустые весы показывают небольшое отклонение.

### Когда не устанавливать ноль

Не устанавливайте ноль:

- когда на весах находится вагон или груз;
- во время незавершенного взвешивания вагона;
- если показание сильно отличается от нуля;
- если программа показывает предупреждение о превышении лимита.

### Порядок установки

1. Убедитесь, что весы пустые.
2. Убедитесь, что показание близко к нулю.
3. Нажмите `Ноль`.
4. Проверьте, что значение стало `0.00` или близким к нему.

Если программа показывает предупреждение, ноль не установлен.

## 9. Корректировки и перенос

Раздел `Корректировки` используется для переноса локальных результатов в учетную
систему и исправления уже перенесенных записей.

### Структура окна

В окне есть:

- фильтр по составу или суткам;
- список `Не перенесённые`;
- список `Перенесённые`;
- верхняя панель реквизитов выбранной записи;
- кнопки `Перенести`, `Сохранить`, `Очистить`, `Обновить`.

### Фильтр записей

- `Состав` - показывать записи по дате и времени состава.
- `Сутки` - показывать записи за выбранную дату.

После изменения фильтра нажмите `Обновить`.

### Поля реквизитов

| Поле | Что вводить |
| --- | --- |
| `№ вагона` | Номер вагона |
| `Груз` | Наименование груза |
| `Тара` | Значение тары из списка |
| `Потребитель` | Получатель или потребитель |
| `Плательщик` | Плательщик |
| `Цех` | Код цеха |
| `Приход` | Перенос в `GPRI` |
| `Расход` | Перенос в `GRAS` |
| `Брутто` | Запись полного веса |
| `Тара` | Запись тары |

### Перенос неперенесенной записи

1. Откройте `Корректировки`.
2. Нажмите `Обновить`, если список неактуален.
3. В списке `Не перенесённые` выберите нужный вагон.
4. Проверьте дату, время, режим, порядковый номер и вес.
5. Выберите `Приход` или `Расход`.
6. Введите `№ вагона`.
7. Выберите `Брутто` или `Тара`.
8. Если выбран `Брутто`, заполните груз и выберите тару при необходимости расчета нетто.
9. Если выбрана `Тара`, проверьте, что запись действительно является тарой.
10. Заполните `Потребитель`, `Плательщик`, `Цех`, если это требуется.
11. Проверьте расчет `Брутто`, `Тара`, `Нетто`.
12. Нажмите `Перенести`.

После успешного переноса запись появится в учетной системе. Локальная запись
будет помечена как перенесенная.

### Что проверить перед переносом

- правильность номера вагона;
- правильность направления `Приход` или `Расход`;
- тип записи `Брутто` или `Тара`;
- вес брутто;
- выбранную тару;
- расчет нетто;
- груз;
- потребителя, плательщика и цех.

### Если перенос не выполнен

1. Не нажимайте `Перенести` много раз подряд.
2. Запишите номер вагона, время и текст сообщения.
3. Откройте `Журнал` и проверьте событие ошибки.
4. Сообщите администратору.

Если программа сообщает, что запись перенесена, но локальная запись не помечена
как переданная, не создавайте новую запись вручную. Возможна ситуация, когда
запись уже есть в учетной системе.

## 10. Исправление перенесенных записей

Исправление выполняется в разделе `Корректировки`.

### Когда используется исправление

Исправление требуется, если:

- ошибочно введен номер вагона;
- нужно уточнить груз;
- неверно выбрана тара;
- нужно исправить потребителя, плательщика или цех;
- требуется пересчитать нетто после выбора правильной тары.

### Порядок исправления

1. Откройте `Корректировки`.
2. В списке `Перенесённые` выберите нужную запись.
3. Проверьте, что в верхней панели появились текущие данные.
4. Исправьте нужные поля.
5. Проверьте расчет `Брутто`, `Тара`, `Нетто`.
6. Нажмите `Сохранить`.
7. После обновления списка убедитесь, что запись отображается с новыми данными.

Все изменения записываются в журнал.

### Что нельзя делать без согласования

Не исправляйте без разрешения:

- записи другой смены;
- записи с неясным происхождением;
- записи, по которым уже оформлены документы;
- весовые значения, если нет подтвержденного основания.

## 11. Печать отвесной

Раздел `Печать` используется для поиска записей и формирования PDF-отвесной.

### Поля поиска

| Поле | Назначение |
| --- | --- |
| `Приход (GPRI)` | Искать приходные записи |
| `Расход (GRAS)` | Искать расходные записи |
| `Дата с` / `по` | Период поиска |
| `Груз` | Фильтр по наименованию груза |
| `Вагон` | Фильтр по номеру вагона |
| `Получатель` | Фильтр по потребителю |
| `№ отвесной` | Фильтр по номеру документа |
| `Печатать грузополучателя` | Добавить потребителя в PDF |

### Поиск записей

1. Откройте `Печать`.
2. Выберите `Приход (GPRI)` или `Расход (GRAS)`.
3. Укажите даты `с` и `по`.
4. При необходимости заполните фильтры.
5. Нажмите `Найти`.
6. Проверьте найденные строки.

Если фильтры заполнены слишком строго и строк нет, нажмите `Очистить фильтры` и
повторите поиск.

### Формирование отвесной

1. Выполните поиск.
2. Отметьте строки, которые нужно включить в отвесную.
3. Если ничего не отметить, в печать попадут все найденные строки.
4. Заполните `Номер отвесной`.
5. Заполните `Откуда`.
6. Заполните `Куда`.
7. При необходимости включите `Печатать грузополучателя`.
8. Нажмите `Просмотр печати`.

Программа создаст PDF и откроет его стандартным приложением Windows.

### Перед печатью проверьте

- выбран правильный период;
- выбран правильный тип записей;
- в таблице нет лишних строк;
- номер отвесной указан верно;
- поля `Откуда` и `Куда` заполнены корректно;
- итоги по брутто, таре и нетто соответствуют ожиданию.

## 12. Журнал

Раздел `Журнал` нужен для просмотра действий пользователей и ошибок.

### Что записывается в журнал

В журнал попадают:

- запуск приложения;
- открытие форм;
- сохранение взвешивания;
- завершение состава;
- установка нуля;
- перенос записи;
- исправление записи;
- формирование отвесной;
- сохранение настроек;
- сохранение калибровки;
- ошибки связи, базы данных и PDF.

### Просмотр журнала

1. Откройте `Журнал`.
2. Укажите период `С` и `по`.
3. Нажмите `Найти`.
4. Найдите нужное событие по времени, типу или объекту.

### Выгрузка в CSV

1. Выполните поиск в журнале.
2. Нажмите `Выгрузка в CSV`.
3. Выберите место сохранения файла.
4. Передайте файл ответственному лицу, если требуется разбор ситуации.

## 13. Сервисный раздел

Раздел `Сервис` предназначен для настройки и диагностики. Обычный оператор
должен использовать его только в пределах своей инструкции.

### Канал

На вкладке `Канал` выбирается активный канал:

- `Основной - CH0`;
- `Резервный - CH1`.

Изменение канала применяется сразу. Перед переключением убедитесь, что это
разрешено ответственным лицом.

### Мониторинг

На вкладке мониторинга можно:

- выбрать COM-порт;
- нажать `Подключить`;
- обновить список портов;
- смотреть текущие значения `CH0` и `CH1`;
- включать или отключать лог;
- очищать лог.

Используйте мониторинг при проверке связи и диагностике оборудования.

### Калибровка статики

Вкладка `Калибровка Статика` содержит таблицу точек:

- `Код АЦП`;
- `Масса, т`;
- `Активна`;
- `K/65535`.

Доступные действия:

- `Добавить строку`;
- `Удалить выбранную`;
- `Захватить`;
- расчет коэффициентов;
- `Применить и сохранить`.

Калибровку выполняет только ответственный пользователь. После изменения
калибровки нужно выполнить контрольное взвешивание.

### Калибровка динамики

Вкладка динамической калибровки содержит коэффициенты:

- `K->` для направления `->`;
- `K<-` для направления `<-`.

Для каждого направления можно захватить код АЦП, указать эталонную массу и
рассчитать коэффициент. После проверки нажимается `Применить и сохранить`.

### Настройки

На вкладке `Настройки` доступны:

- `COM-порт`;
- `НПВ (т)`;
- `Дискретность`;
- `Лимит нуля (% НПВ)`;
- `Новый пароль`.

Изменение настроек влияет на работу всех режимов взвешивания. Не меняйте их без
понимания последствий.

## 14. Типовые ошибки и действия

| Ситуация | Возможная причина | Действие |
| --- | --- | --- |
| `АЦП: отключен` | Нет связи с оборудованием | Проверьте питание, кабель, COM-порт |
| Ошибка подключения к порту | Порт занят или выбран неверно | Закройте другие программы, выберите правильный порт |
| Значение веса не меняется | Нет данных от АЦП | Проверьте мониторинг и физическое подключение |
| Нельзя установить ноль | Вес больше разрешенного лимита | Освободите весы или вызовите администратора |
| Не удалось сохранить результат | Недоступна локальная база | Не продолжайте массовое взвешивание, сообщите администратору |
| Не удалось перенести запись | Недоступна учетная база Firebird | Не повторяйте перенос много раз, проверьте журнал |
| Запись перенесена, но локально не помечена | Ошибка локальной базы после вставки в учет | Не создавайте дубль, сообщите администратору |
| Превышен максимально допустимый порог | Вес выше НПВ | Не переносите запись, разберите ситуацию |
| Не удалось сформировать PDF | Ошибка генерации или открытия файла | Повторите печать, при повторной ошибке сообщите администратору |
| В печати нет строк | Неверный период или фильтр | Очистите фильтры и повторите поиск |
| Нет вариантов тары | По вагону нет истории тары | Проверьте номер вагона или внесите данные по регламенту |
| Неверный расчет нетто | Неверно выбрана тара | Выберите правильную тару и пересчитайте |

## 15. Контрольные листы

### Перед сменой

- Оборудование включено.
- Кабель связи подключен.
- Программа запущена.
- Индикатор АЦП зеленый.
- Весы пустые.
- Ноль проверен.
- Режим взвешивания выбран правильно.
- При динамике проверено направление.

### Перед переносом

- Выбрана правильная локальная запись.
- Вес не превышает допустимый предел.
- Номер вагона введен верно.
- Выбрано `Приход` или `Расход`.
- Выбрано `Брутто` или `Тара`.
- Тара выбрана корректно.
- Нетто проверено.
- Реквизиты заполнены.

### Перед печатью

- Выбран правильный период.
- Выбран правильный тип записей.
- Фильтры не исключают нужные строки.
- Выбраны нужные строки.
- Номер отвесной указан.
- `Откуда` и `Куда` заполнены.
- PDF открылся и визуально проверен.

## 16. Правила безопасности данных

- Не повторяйте перенос несколько раз подряд при ошибке.
- Не исправляйте записи, если не уверены в основании исправления.
- Не меняйте калибровку во время обычной работы смены.
- Не устанавливайте ноль при занятой платформе.
- Не игнорируйте красный индикатор АЦП.
- Не закрывайте программу во время незавершенного взвешивания без необходимости.
- При спорной ситуации сначала проверьте журнал.

## 17. Что сообщать администратору

При обращении к администратору передайте:

- дату и время проблемы;
- режим работы: статика, динамика, перенос, печать, сервис;
- номер вагона, если он известен;
- номер состава или время состава;
- текст сообщения об ошибке;
- что было сделано перед ошибкой;
- была ли запись уже перенесена;
- есть ли событие в журнале.

Чем точнее описание, тем меньше риск дублей и неправильных исправлений.
