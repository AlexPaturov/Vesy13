# База данных

## PostgreSQL: локальная база

Локальная база используется для хранения калибровки, временного журнала
взвешиваний и аудита.

Фактическое подключение сейчас находится в коде `LocalRepository` и
`AuditLogger`.

### `wagon_weighing`

Хранит результаты потележечного взвешивания до и после переноса.

| Поле | Назначение |
| --- | --- |
| `id` | первичный ключ |
| `train_time` | время начала состава |
| `wagon_time` | время фиксации вагона |
| `wagon_num` | порядковый номер вагона в составе |
| `bogie1` | вес первой тележки, т |
| `bogie2` | вес второй тележки, т |
| `total` | общий вес, т |
| `direction` | направление для динамического режима |
| `mode` | `СТАТИКА` или `ДИНАМИКА` |
| `transferred` | признак переноса в Firebird |
| `when_insert` | время вставки |

### `calibration_points`

Хранит статическую калибровку по каналам.

| Поле | Назначение |
| --- | --- |
| `id` | первичный ключ |
| `channel` | канал АЦП: `0` или `1` |
| `adc_code` | код АЦП |
| `mass` | масса в тоннах |
| `is_active` | участвует ли точка в расчете |
| `deleted_at` | мягкое удаление |

При сохранении точек неактивные записи не удаляются физически: им выставляется
`deleted_at`.

### `calibration_dynamic`

Хранит коэффициенты динамического режима:

- `k_plus` - коэффициент для направления `->`;
- `k_minus` - коэффициент для направления `<-`;
- `updated_at` - время обновления.

В коде используется одна строка с `id = 1`.

### `audit_log`

Хранит события приложения и ошибки:

- открытие форм;
- сохранение взвешивания;
- перенос и корректировки;
- генерация отвесной;
- сохранение настроек и калибровки;
- подключение/ошибки АЦП;
- ошибки БД/PDF/общие ошибки.

`LogsForm` читает до 5000 записей за выбранный период и поддерживает экспорт в
CSV.

## Firebird: заводская база

`FactoryRepository` работает с таблицами:

- `GPRI` - приход;
- `GRAS` - расход.

Основные операции:

- `InsertAsync` - вставляет перенесенную локальную запись.
- `UpdateAsync` - обновляет существующую запись.
- `GetRecentAsync` - читает последние записи из обеих таблиц.
- `GetByTrainTimeAsync` и `GetByDateAsync` - ищут записи по времени состава или дате.
- `GetForPrintAsync` - выбирает записи для отвесной по фильтрам.
- `GetTaraOptionsAsync` - получает варианты тары за последний год по номеру вагона.

## Сопоставление локальной записи и Firebird

При переносе из `LocalWagon` создается `GpriGras`:

| Firebird поле | Источник |
| --- | --- |
| `Table` | выбор оператора: `GPRI` или `GRAS` |
| `DT` | дата `WagonTime` |
| `VR` | время `TrainTime` |
| `VR_PRV` | время `WagonTime` |
| `NVAG` | номер вагона, введенный оператором |
| `NPP` | `LocalWagon.Number` |
| `REJVZVESH` | режим взвешивания |
| `BRUTTO` | общий вес, если это брутто |
| `TAR_BRS` | общий вес, если это тара |
| `NETTO` | `BRUTTO - TAR_BRS`, если выбрана тара |
| `GRUZ`, `POTR`, `PLAT`, `CEX` | реквизиты из формы корректировок |

После успешной вставки в Firebird локальная запись помечается
`transferred = true`.
