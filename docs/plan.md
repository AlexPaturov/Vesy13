# План разработки ПО «Vesy13»

## Текущее состояние
- WinForms, C# (.NET 8)
- COM-порт: поллинг, приём байт, детекция фрейма по таймауту 20 мс
- `OnFrameReceived` — пустой TODO
- Протокол команд задокументирован в `protocol.md`

---

## Архитектура форм

### `MainForm` — главное меню
Картинка (логотип/фон) и панель с пунктами меню:
- **Взвешивание** → открывает `WeighingForm`
- **Сервис** → запрос пароля (`PasswordDialog`) → открывает `ServiceForm`
- **Корректировки** → открывает `CorrectionsForm`
- **Печать отвесной** → открывает `PrintForm`
- **Просмотр логов** → открывает `LogForm`

  Минимум элементов, работает как лаунчер:
  - Логотип / фоновое изображение
  - 5 кнопок-пунктов меню (крупные, видно с расстояния):
    - Взвешивание
    - Сервис
    - Корректировки
    - Печать отвесной
    - Просмотр логов
  - Индикатор связи с АЦП (есть / нет) — виден всегда, в углу
  - Версия ПО — мелко внизу

---

### `WeighingForm` — экран взвешивания
- Переключатель режима **Статика / Динамика**
- Выбор канала **Основной / Резервный**
- Выбор направления **+ / –** (только Динамика, обязательно)
- Крупный дисплей текущей массы
- Индикатор состояния: **Тележка 1 → Тележка 2 → Вагон**
- Одна рабочая кнопка фиксации веса: «Взвесить» / «Сохранить» / «Записать» / **Пробел**
- 1-е нажатие фиксирует 1-ю тележку во внутренней памяти сессии
- 2-е нажатие фиксирует 2-ю тележку, суммирует вагон и сохраняет его в локальную БД
- Кнопка «Ноль» (только Статика)
- Индикатор связи с АЦП
- Индикатор превышения НПВ

  Верхняя панель — выбор режима:
  - RadioButton / ToggleButton: Статика | Динамика
  - RadioButton: Основной | Резервный (канал)
  - RadioButton: → | ← (направление, только Динамика, обязательно до начала)

  Центр — дисплей массы (крупно):
  - Текущая масса — большой шрифт (60–80pt)
  - Единица: т
  - Цвет: зелёный → норма, красный → превышение НПВ

  Статус-строка под дисплеем:
  - Индикатор состояния сессии: Тележка 1 → Тележка 2 → Вагон
  - Результаты по тележкам и итог вагона (после взвешивания)

  Кнопки действий:
  - Взвесить / Сохранить / Записать (Пробел) — основная кнопка, крупная
  - 1-е нажатие — фиксация тележки 1 в памяти
  - 2-е нажатие — фиксация тележки 2, суммирование и сохранение вагона в БД
  - Ноль — только Статика

  Нижняя панель:
  - Индикатор связи с АЦП
  - Кнопка «Назад» → MainForm

---

### `ServiceForm` — сервисный режим (по паролю)
- Мониторинг CH0 и CH1 одновременно (коды АЦП + масса)
- Калибровка Статика (таблица точек)
- Калибровка Динамика (коэффициенты «+» / «–»)
- НПВ, дискретность, лимит нуля
- Фильтр усреднения
- Выбор COM-порта
- Кнопка «Сохранить»

Вкладки (TabControl):

Вкладка «Мониторинг»:
- Оба канала одновременно: код АЦП CH0, код АЦП CH1, масса CH0, масса CH1 — обновляется в реальном времени
- Индикатор связи, частота фреймов

Вкладка «Калибровка Статика»:
- Выбор канала (CH0 / CH1)
- Таблица калибровочных точек (код АЦП ↔ масса)
- Кнопки: Добавить, Удалить, Рассчитать
- Поля k и b — ручной ввод / результат расчёта

Вкладка «Калибровка Динамика»:
- Коэффициент направления «→» и «←»
- Ввод эталонной массы → авторасчёт коэффициента
- Ручной ввод коэффициентов

Вкладка «Настройки»:
- Выбор COM-порта из списка
- НПВ (т)
- Дискретность: 0.1 / 0.05 / 0.01 т
- Лимит установки нуля (% от НПВ)
- Фильтр усреднения АЦП
- Пароль (смена)
- Кнопка Сохранить

---

### `CorrectionsForm` — корректировки
- Работа с вагонами, уже переданными в систему учёта предприятия
- Поиск/выборка ранее взвешенных вагонов
- Ручная корректировка результатов взвешивания с передачей исправленных данных обратно в систему учёта
- **Интеграция:** отдельная БД системы учёта предприятия (строка подключения и структура таблиц — будут предоставлены позже)

### `PrintForm` — печать отвесной
- Фильтры для выборки из системы учёта предприятия (дата, номер вагона и т.д.)
- Окно просмотра результатов выборки
- Выбор вагонов для печати через чекбоксы
- Предпросмотр и печать отвесной по выбранным вагонам
- **Интеграция:** та же БД системы учёта предприятия (строка подключения и структура таблиц — будут предоставлены позже)

### `LogForm` — просмотр логов
- Таблица: дата, время, режим, канал, направление, коды АЦП, масса, событие
- Фильтр по дате
- Кнопка «Экспорт CSV»

### `PasswordDialog` — диалог пароля
- Модальный диалог, вызывается перед входом в `ServiceForm`

## Этап 1. Парсинг фрейма АЦП

**Цель:** получить из сырых байт числовое значение кода АЦП по каждому каналу.

Задачи:
- ~~Изучить формат ответного фрейма~~ **Формат известен:**
  - Длина всегда **4 байта**, контрольной суммы нет
  - `CH0 = byte[1] * 256 + byte[0]`
  - `CH1 = byte[3] * 256 + byte[2]`
  - COM-порт: 4800 бод, **чётность Even**, 8 бит, 1 стоп-бит
- Реализовать `FrameParser` — статический класс, принимающий `byte[]`, возвращающий структуру `AdcFrame { int Ch0, int Ch1, bool Valid }`
- Покрыть парсер unit-тестами (xUnit)
- В `OnFrameReceived` вызывать парсер и передавать результат в модель

## Этап 2. Настройки и конфигурация

**Цель:** хранить все параметры в файле, загружать при старте.

Задачи:
- Создать `Settings.cs` (JSON-сериализация через `System.Text.Json`):
  - COM-порт, скорость, режим АЦП (Static/Dynamic), канал (Main/Backup)
  - НПВ, дискретность (0.1 / 0.05 / 0.01 т)
  - Лимит нуля (% от НПВ)
  - Пароль сервисного режима (hash)
  - Калибровочные коэффициенты (см. Этап 5)
  - Настройки усреднения/фильтра
- `SettingsService` — загрузка/сохранение `settings.json` рядом с exe
- Форма настроек COM-порта (выбор порта из списка, скорость)

## Этап 3. Модель данных взвешивания

**Цель:** чистый бизнес-слой, независимый от UI.

Задачи:
- `WeighingSession` — сессия взвешивания:
  - Режим (Static / Dynamic)
  - Направление (Plus / Minus) — только Dynamic
  - Список `BogieRecord { DateTime Time, double RawCode, double MassKg }`
  - Свойство `WagonMassKg` — сумма двух тележек
  - Состояние: `WaitingBogie1 → WaitingBogie2 → Done`
- `ZeroService` — установка нуля, хранение смещения по каждому каналу
- `Overload` — проверка превышения НПВ

---

## Этап 4. Режим «Статика»

**Цель:** взвешивание стоящего вагона.

Задачи:
- Переключение АЦП в Static-режим при старте (байт команды из таблицы `protocol.md`, Avg=256, CH0+CH1)
- UI: кнопка «Взвесить» (или Пробел), поле отображения текущей массы
- Усреднение: накапливать коды АЦП за настраиваемый интервал → среднее → применить калибровку
- Установка нуля: кнопка «Ноль», проверка, что текущее значение в пределах лимита (% от НПВ)
- Индикация связи: если фреймы не приходят >2 с — «Нет связи» красным
- Превышение НПВ: цвет + звук + блокировка сохранения

---

## Этап 5. Калибровка

**Цель:** перевод кода АЦП в тонны.

### 5а. Статическая калибровка
- Многоточечная: список точек `(code, massKg)`, метод наименьших квадратов → линейный коэффициент `k` и смещение `b`
- UI в сервисном режиме: таблица точек, добавление/удаление, кнопка «Рассчитать»
- Ручной ввод `k` и `b`
- Независимо для CH0 и CH1

### 5б. Динамическая калибровка
- Коэффициент направления: `kPlus` (для «+»), `kMinus` (для «–»)
- Программный расчёт: ввести эталонную массу → авто-вычисление коэффициента
- Ручной ввод коэффициентов
- Независимо для каждого направления

---

## Этап 6. Режим «Динамика»

**Цель:** взвешивание двух тележек на ходу.

Задачи:
- Переключение АЦП в Dynamic-режим (байт команды из таблицы)
- UI: выбор направления «+» / «–» до начала (обязательно)
- Индикатор текущего состояния: «Тележка 1» → «Тележка 2» → «Вагон»
- Нажатие Пробел / кнопки — старт окна сбора данных; длительность окна задается внутренней константой программы:
  - Накапливать коды АЦП в течение окна
  - Усреднить → применить калибровку + коэффициент направления → `BogieRecord`
- После второй тележки: отображение суммарного веса вагона и автоматическое сохранение вагона в локальную БД
- Если сессия прервана на одной тележке — предупреждение
