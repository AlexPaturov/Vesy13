# Конфигурация и запуск

## Требования

- Windows.
- .NET SDK/Runtime 8 с Windows Desktop.
- PostgreSQL на локальной машине или доступный по сети.
- Firebird-клиент и доступ к заводской базе `DC_OPER.FDB`.
- COM-порт с АЦП "СИМ А04" либо тестовая пара виртуальных COM-портов.


## COM-порты по окружениям

На проде приложение работает с реальным контроллером АЦП через `COM1`.

На dev-стенде основное приложение работает через `COM3`, а эмулятор
`ScaleListener` открывает `COM4`. В системе должна быть установлена программа
для проброса/виртуальной пары COM-портов, связывающая `COM3` и `COM4`.

Перед проверкой связи с АЦП нужно убедиться, что в `settings.json` выбран порт
текущего окружения:

- prod: `AdcPortName = COM1`;
- dev: `AdcPortName = COM3`;
- эмулятор `ScaleListener`: `COM4`.

## Сборка

```bash
dotnet build Vesy13.sln
```

Основное приложение:

```bash
dotnet run --project Vesy13.csproj
```

Тестовый эмулятор:

```bash
dotnet run --project ScaleListener/ScaleListener.csproj
```

## `settings.json`

`SettingsService` ищет файл `settings.json` рядом с исполняемым файлом. Если
файла нет или он поврежден, создается новый с настройками по умолчанию.

Поля:

| Поле | Назначение | Значение по умолчанию |
| --- | --- | --- |
| `AdcPortName` | COM-порт АЦП | prod: `COM1`, dev: `COM3` |
| `MaxCapacityTonnes` | НПВ, максимальная грузоподъемность в тоннах | `140.0` |
| `WeightDiscretizationTonnes` | шаг округления веса | `0.05` |
| `OperatorZeroLimitPercent` | предел установки нуля оператором, % от НПВ | `2.0` |
| `AdminZeroLimitPercent` | административный предел установки нуля, % от НПВ | `100.0` |
| `AdminPasswordHash` | PBKDF2-хеш пароля администратора | создается при первом запуске |
| `AdminPasswordSalt` | соль пароля администратора | создается при первом запуске |
| `CachedStaticPoints` | локальный снимок всех точек статической калибровки (полные строки из БД, включая снятые) — fallback на случай недоступности БД при старте | обновляется при каждой успешной загрузке/сохранении калибровки |
| `CachedDynamicCalib` | локальный снимок активной строки динамической калибровки — тот же fallback | обновляется вместе с `CachedStaticPoints` |
| `CalibCacheUpdatedAt` | время последнего обновления двух полей выше | `null`, пока калибровка ни разу не была успешно прочитана |

Фильтры входного потока (подробности — `docs/hardware.md`, раздел «Обработка входного
потока»). Правятся на вкладке `Настройки` формы `ServiceForm`, применяются без перезапуска.
По умолчанию выключены все, границы клэмпа инертны — реальные подбираются на живом железе:

| Поле | Назначение | Значение по умолчанию |
| --- | --- | --- |
| `StaticClampEnabled` | клэмп кодов статики | `false` |
| `StaticClampMinCode` | нижняя граница клэмпа статики | `0` |
| `StaticClampMaxCode` | верхняя граница клэмпа статики | `65535` |
| `StaticDeltaEnabled` | дельта-фильтр статики | `false` |
| `StaticDeltaMaxCodes` | макс. допустимый скачок кода статики | `5000` |
| `StaticEmaEnabled` | EMA-сглаживание статики | `false` |
| `StaticEmaAlpha` | коэффициент `α` EMA статики, `(0..1]` | `0.3` |
| `DynamicClampEnabled` | клэмп кодов динамики | `false` |
| `DynamicClampMinCode` | нижняя граница клэмпа динамики | `0` |
| `DynamicClampMaxCode` | верхняя граница клэмпа динамики | `65535` |
| `DynamicDeltaEnabled` | дельта-фильтр динамики | `false` |
| `DynamicDeltaMaxCodes` | макс. допустимый скачок кода динамики | `5000` |
| `DynamicStuckEnabled` | детектор застревания динамики | `false` |
| `DynamicStuckSamples` | сколько строго равных кодов подряд считать застреванием (≈5 с при 30 Гц) | `150` |
| `DynamicEmaEnabled` | EMA-сглаживание динамики | `false` |
| `DynamicEmaAlpha` | коэффициент `α` EMA динамики, `(0..1]` | `0.2` |

`SettingsService.EnsureDefaults` чинит выходящие за границы значения молча, возвращая их к
дефолту: `Max <= Min` у клэмпа сбрасывает обе границы, `DeltaMaxCodes <= 0` → `5000`,
`DynamicStuckSamples < 2` → `150`, `EmaAlpha` вне `(0..1]` → `0.3`/`0.2`. Те же границы
проверяет форма `Настройки` перед сохранением.

После первого запуска пароль администратора нужно сменить через сервисную форму.

Если БД недоступна при старте, `LocalRepository.RestoreLastKnownCalibration` подставляет
`CachedStaticPoints`/`CachedDynamicCalib` вместо пустой калибровки; событие пишется в аудит
(`AuditLogger.CalibrationFallback`). Пустой кэш (калибровка никогда не была успешно прочитана)
оставляет то же заблокированное состояние, что и обычная неудача чтения из БД.

## Строки подключения

Сейчас строки подключения заданы в коде:

- PostgreSQL:
  - `LocalRepository`
  - `AuditLogger`
  - база: `scale_db`
  - пользователь: `scale_user`
- Firebird:
  - `FactoryRepository`
  - база: `DC_OPER.FDB`
- Application Insights:
  - `AuditLogger`

Для эксплуатации на нескольких стендах лучше вынести эти значения в
конфигурационный файл, переменные окружения или защищенное хранилище.

## Первый запуск

1. Создать PostgreSQL-базу и пользователя согласно фактической строке
   подключения.
2. Выполнить миграции из `migrations/` по порядку:
   - `migration_v2.sql`
   - `002_calibration_points.sql`
   - `003_calibration_points_deleted_at.sql`
   - `004_calibration_dynamic_history.sql`
   - `005_calibration_points_created_at.sql`
3. Запустить приложение.
4. Открыть сервисную форму.
5. Проверить COM-порт и сохранить настройки.
6. Выполнить калибровку основного и резервного каналов.
7. Для динамического режима задать коэффициенты движения `+` и `-`.
8. Сменить пароль администратора.

## Настройки COM-порта

`SimA04ReaderStatic` открывает порт с фиксированными параметрами:

- скорость: `4800`;
- parity: `Even`;
- data bits: `8`;
- stop bits: `One`;
- write timeout: `500 ms`;
- read timeout: бесконечный.

Имя порта берется из `settings.json`.
