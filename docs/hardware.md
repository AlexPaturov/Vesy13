# Аппаратная интеграция

## АЦП "СИМ А04"

Основной драйвер находится в `Services/Hardware/SimA04Reader.cs`.

Драйвер работает по циклу:

1. открыть COM-порт;
2. отправить один poll-байт;
3. принять ответ;
4. подождать 20 мс после последнего принятого байта;
5. разобрать накопленный буфер как кадр;
6. опубликовать события;
7. отправить следующий poll-байт.

## Параметры порта

- baud rate: `4800`;
- parity: `Even`;
- data bits: `8`;
- stop bits: `One`;
- poll byte по умолчанию: `214`;
- ожидаемый ответ: 4 байта.

## Формат кадра

Кадр АЦП представлен моделью `SimA04Frame`.

Ожидаемая длина - ровно 4 байта. Если длина отличается, кадр считается
невалидным.

Расчет каналов:

```text
CH0 = byte[1] * 256 + byte[0]
CH1 = byte[3] * 256 + byte[2]
```

Каналы используются так:

- `ActiveChannel.Main` - основной канал, `CH0`;
- `ActiveChannel.Backup` - резервный канал, `CH1`.

## События `SimA04Reader`

- `FrameReceived` - валидный распарсенный кадр.
- `RawFrameReceived` - сырые байты, используются для диагностики в сервисной форме и аудите.
- `ConnectionChanged` - изменение состояния COM-соединения.

## Статическая калибровка

Статический расчет выполняется по активным точкам текущего канала.

Алгоритм:

1. выбрать активные точки канала;
2. отсортировать по `adc_code`;
3. выбрать последнюю точку, код которой меньше или равен текущему коду;
4. если код ниже первой точки, использовать первую;
5. если код выше последней, использовать последнюю;
6. посчитать вес как:

```text
тонны = текущий_код * (масса_точки / код_точки)
```

Интерполяция между точками не используется.

## Динамическая калибровка

Динамический режим использует коэффициенты из `calibration_dynamic`:

```text
тонны = K * код_АЦП
```

Выбор коэффициента зависит от направления:

- направление начинается с `->` - `k_plus`;
- иначе - `k_minus`.

## Установка нуля

Оператор может установить ноль только если текущий вес находится в пределах
`OperatorZeroLimitPercent` от НПВ. Предел в тоннах вычисляется как:

```text
MaxCapacityTonnes * OperatorZeroLimitPercent / 100
```

Смещение нуля хранится в памяти формы взвешивания и применяется к текущим
показаниям до округления.

## Эмулятор `ScaleListener`

Проект `ScaleListener` нужен для тестирования обмена по COM-порту.

Особенности:

- открывает `COM4`;
- принимает poll-байт `214`;
- генерирует 4-байтовый ответ для двух каналов;
- переводит заданный вес в код АЦП по диапазону `0..65535`;
- может добавлять случайную погрешность в тоннах;
- показывает лог входящих и исходящих байтов.

Для локального тестирования обычно нужна пара виртуальных COM-портов: один порт
указывать в `settings.json` основного приложения, второй - использовать
эмулятором.
